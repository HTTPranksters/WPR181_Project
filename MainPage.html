The World Wide Web (WWW) and the Internet have transformed the way we communicate, access information, and conduct business. This document delves into the intricacies of the World Wide Web and the evolution of the Internet, exploring their history, differences, protocols, architecture, and engineering aspects.

Overview of the World Wide Web

The World Wide Web (WWW) was created by Tim Berners-Lee in 1989. Tim was a British computer scientist for CERN (European Organization for Nuclear Research) in Switzerland and is now often referred to as the father of the web. The motivation behind creating the WWW was to solve the challenges experienced by the researchers at CERN, like sharing research and accessing this research. At that time researchers and scientists as well as institutions were generating large amounts of documents and data and they needed a better way of sharing, organising, and accessing this data. So, in 1989 Tim developed the first web browser at CERN for better communication and information sharing between scientists. It was called the World Wide Web, a browser, and an editor. His vision included linking documents by using hyperlinks and the usage of protocols for this communication, like Hyper Text Transfer Protocol (HTTP). (Britannica, 2023). The world wide web is a collection of all websites and pages that allows users to connect their local devices to other nonlocal devices through the internet. The world wide web can show information on website in many ways such as text, audio, images, and video. It allows this communication around the world using hypertext and HTTP.

WWW vs the Internet

The World Wide Web is a virtual system that allows information and hypermedia to be shared over the Internet through hyperlinks that are referenced by Uniform Resource Locators (URLs). Users search up a word or phrase and can access documents on that specific word or phrase and other documents related to that word or phrase.

The Internet is a network system that provides worldwide communication and access to data resources including videos, audio, and images. It’s governed by agencies that establish protocols.

Here are some differences between the two:

  • Internet uses IP addresses while the World Wide Web uses HTTP.
  • World Wide Web is part of the Internet.
  • Internet is hardware-based while the World Wide Web is software-based.
  • Internet originated in the late 1960s while the World Wide Web was established in 1989.
  • Internet connects a device to other devices irrespective of their location(s), while the World Wide Web contains information and makes it accessible through the Internet.
  • The Internet is like a bookstore and the Web is the collection of books in the store.

Internet protocols

Internet protocols are the roles that governs the exchange of data over the internet. Both the device that sends the data and the device that is receiving the data are required to follow the same protocols for the communication of data to be successfully. When data is transferred over the internet it is sent in small packets of organized data. The internet protocols define how those packets are organized.

They include the following:

  • TCP/IP (Transmission Control Protocol/Internet Protocol): This is the foundational protocol suite of the internet. It is used to ensure reliable communication by breaking data down into packets, sending them to their destination and the reassembling them.
  • HTTP/HTTPS (Hypertext Transfer Protocol/Secure): This protocol is used to for transferring web pages from web servers to web browsers. It also defines the way how data is requested and presented on the WWW. HTTPS is just the secure version of HTTP, which uses encryption to protect data and provide security and privacy.
  • FTP (File Transfer Protocol): This protocol is used when transferring files over the internet. It provides users with the opportunity to upload and download files to and from remote servers.
  • DNS (Domain Name System): This protocol is responsible for converting domain names that humans can read, like (www.example.com) into IP-addresses computers can understand, It gives users the opportunity to access the web by using domain names instead of IP-addresses.
  • SMTP/POP3/IMAP (Simple Mail Transfer Protocol/Post Office Protocol/Internet Message Access Protocol): These protocols are used in email communication. SMTP is responsible for sending emails, while POP3 and IMAP is responsible for receiving emails on the client’s side. POP3 is used to download emails and remove them from the server, while IMAP keeps the emails on the server and just synchronises them with the client.

Web Application Architecture

Web Application Architecture refers to the components are organised, forming the structure of a web application. It also includes how components communicate with each other to provide a user experience. Developers use different architectural patterns to develop web applications, these are some of the different web application architecture patterns:

Client-Server Architecture: This is a fundamental model where the application is divided into two main components, which include the client and server. The client, usually a web browser, requests data from the server and the server receives these requests, processes them, and then sends back the requested data to the client. The client side handles the user interface, and the server side can be optimized for data processing.

Three-Tier Architecture: When using this architecture, a web application gets divided into the following three layers: presentation (client), application logic (server) and storage (database). This modular structure increases scalability and maintainability. When using this structure the client communicates with the application server, which then processes these requests and then communicates with the database server to receive data or store data.

Client-side and server-side engineering

Client-side engineering involves creating the user interface and experience within web browsers using HTML, CSS, and JavaScript.Server-side engineering focuses on handling requests, processing data, and managing databases using languages like Python, Ruby, PHP, and frameworks like Node.js and Django.

Thus, the evolution of the Internet and the development of the World Wide Web have revolutionized how we communicate and access information. Understanding the history, protocols, and architecture of these technologies is crucial in today's interconnected world.
